Developer quickstart

Build a durable AI workflow

01 / What you are building

Workflows move AI pipelines from a script to a durable system.

Your code stays in your environment. Mistral handles orchestration, state, retries, and observability so multi-step work can survive crashes and long waits.

Local codeThe worker runs where your project, dependencies, and secrets already live.
Managed stateThe platform tracks workflow runs and resumes from completed activities.
Visible executionRuns can be launched and watched from Console or from the Makefile.

Source: docs.mistral.ai / Build a workflow

04 / The workflow contract

Activities are durable steps; the workflow definition is the triggerable unit.

If the process crashes mid-run, Mistral can replay from the last completed activity.
@workflows.activity()
async def greet(name: str) -> str:
    return f"Hello, {name}!"

@workflows.workflow.define(
    name="hello-world"
)
class HelloWorkflow:
    @workflows.workflow.entrypoint
    async def run(self, input):
        return await greet(input.name)

The worker is the bridge between local execution and managed orchestration.

06 / Trigger and verify

Keep the worker open, then start a run from a second terminal or Console.

make start-worker

make execute workflow=hello-world input='{"name":"World"}'

  "result": "Hello, World! Welcome to Mistral Workflows."
}

TroubleshootingIf the worker shows a connection error, confirm the API key and that the worker is still running.

07

07 / What to remember

The minimal workflow is small, but the operating model is production-shaped.

DefineName the workflow so it can be registered and triggered.
IsolateMark durable steps with the activity decorator.
RunUse the worker as the long-lived task executor.
ObserveFollow executions from Console and verify outputs.
